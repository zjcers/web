Personal Styleguide

C

Lately, I've been following the Linux Kernel Coding Style. This is a departure from how I used to write C, so my projects will be in limbo for a while.

C++

I use the same styling rules as C for the most part. I'll update this section once I get back into a C++ project again.

Python

PEP8 says that I should use 4-space tabs. It says a lot of other things, too. A lot of those I don't care to read or just ignore. Like the recommendation to use underscore- deliminated variable names. I like camelCase too much for that. It looks better.

JavaScript

Example

(From Linter-CNew) 
				
buildCommand: function(activeEditor, file) {
  if (atom.config.get("linter-gcc.gccDebug")) {
    console.log("linter-gcc config: " + JSON.stringify(settings));
  }
  let command = this.getCommandFromCMake(file, activeEditor, settings);
  if (!command) {
    /*buildCommandFromSettings returns a complete object*/
    return this.buildCommandFromSettings(file, activeEditor, settings);
  }
  /*Split the binary name from before the first space and
    the args from behind it*/
  let binary = command.substring(0, command.indexOf(" "));
  let args = command.substring(command.indexOf(" ")+1, command.length);
  args = this.fixFlags(args);
  return {binary: this.findBinary(binary), args: args};
},

I try to write JavaScript like I write C/C++ as much as possible. The comment and indentation styles I use are mostly the same. I like the new ES6 variable statements (let and const). They make scoping a lot easier to track. On the other hand, the new short function statements (<varname> =>...) aren't very readible and I avoid using them.

My only big style rules for HTML are to always use tabs to show how many tags a given position is inside and to be as semantic as possible. HTML has a lot of special tags that mean special things. Use them. Like all other languages, I like to keep it legible and cut my line length down to where my editor usually ends (around character 125).
